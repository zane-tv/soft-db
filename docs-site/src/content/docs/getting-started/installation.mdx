SoftDB ships as a native desktop app for all three major platforms. Pick your platform below and you'll be up and running in a few minutes.

## Download

Grab the latest release from the [GitHub Releases page](https://github.com/zane-tv/soft-db/releases).

| Platform | File | Notes |
|----------|------|-------|
| Windows | `SoftDB-amd64-installer.exe` | NSIS installer with shortcuts and uninstaller |
| macOS | `SoftDB-darwin-arm64.dmg` | Apple Silicon native; Intel Macs via Rosetta 2 |
| Linux | `SoftDB-linux-amd64.AppImage` | Portable — no install needed |
| Linux (Debian/Ubuntu) | `SoftDB-linux-amd64.deb` | System package with desktop integration |

## Platform Instructions

<Tabs>
  <TabItem label="Windows">
    <Steps>
      1. Download `SoftDB-amd64-installer.exe` from the Releases page.

      2. Run the installer. Windows may show a SmartScreen warning for unsigned apps — click **More info** then **Run anyway**.

      3. Follow the setup wizard. SoftDB installs to `%LOCALAPPDATA%\SoftDB` by default and adds a Start Menu shortcut.

      4. Launch SoftDB from the Start Menu or desktop shortcut.
    </Steps>

    To uninstall, use **Add or Remove Programs** in Windows Settings.
  </TabItem>

  <TabItem label="macOS">
    <Steps>
      1. Download `SoftDB-darwin-arm64.dmg` from the Releases page.

      2. Open the `.dmg` file and drag **SoftDB** to your Applications folder.

      3. On first launch, macOS may block the app because it's from an unidentified developer. Open **System Settings → Privacy & Security** and click **Open Anyway**.

      4. Launch SoftDB from Applications or Spotlight.
    </Steps>

    Alternatively, install via Homebrew:

    ```bash
    brew tap zane-tv/softdb
    brew install --cask softdb
    ```
  </TabItem>

  <TabItem label="Linux">
    **AppImage (any distro):**

    <Steps>
      1. Download `SoftDB-linux-amd64.AppImage`.

      2. Make it executable:
         ```bash
         chmod +x SoftDB-linux-amd64.AppImage
         ```

      3. Run it:
         ```bash
         ./SoftDB-linux-amd64.AppImage
         ```
    </Steps>

    **Debian/Ubuntu (.deb):**

    <Steps>
      1. Download `SoftDB-linux-amd64.deb`.

      2. Install with dpkg:
         ```bash
         sudo dpkg -i SoftDB-linux-amd64.deb
         ```

      3. Launch from your application menu or run `softdb` in a terminal.
    </Steps>
  </TabItem>
</Tabs>

## Verify Downloads

All release binaries are signed with [Sigstore](https://sigstore.dev) for supply chain security. Each file has a corresponding `.bundle` signature file on the Releases page.

<Steps>
  1. **Install cosign:**
     ```bash
     # macOS
     brew install cosign

     # Go (any platform)
     go install github.com/sigstore/cosign/v2/cmd/cosign@latest
     ```

  2. **Verify the downloaded file:**
     ```bash
     cosign verify-blob SoftDB-amd64-installer.exe \
       --bundle SoftDB-amd64-installer.exe.bundle \
       --certificate-identity-regexp "https://github.com/zane-tv/soft-db" \
       --certificate-oidc-issuer https://token.actions.githubusercontent.com
     ```

     Replace `SoftDB-amd64-installer.exe` with the filename you downloaded. The same command works for `.dmg`, `.AppImage`, and `.deb` files.

  3. **Or use the helper script** (if you cloned the repo):
     ```bash
     ./scripts/verify-release.sh SoftDB-amd64-installer.exe
     ```
</Steps>

A successful verification prints `Verified OK`.

## Build from Source

If you want to run the latest development version or contribute to SoftDB, you can build it yourself.

**Prerequisites:**

- [Go 1.22+](https://golang.org/dl/)
- [Bun](https://bun.sh/) (or Node.js 18+)
- [Wails CLI v3](https://v3.wails.io/getting-started/installation/)

<Steps>
  1. **Install the Wails CLI:**
     ```bash
     go install github.com/wailsapp/wails/v3/cmd/wails3@latest
     ```

  2. **Clone the repository:**
     ```bash
     git clone https://github.com/zane-tv/soft-db.git
     cd soft-db
     ```

  3. **Start development mode** (hot-reload for both Go and React):
     ```bash
     wails3 dev
     ```

  4. **Build a production binary:**
     ```bash
     wails3 build
     ```

     The compiled app lands in the `build/bin/` directory.
</Steps>

<Aside type="note">
  Development mode starts a local Vite dev server for the frontend and recompiles the Go backend on file changes. You'll see both the app window and live reload working together.
</Aside>
